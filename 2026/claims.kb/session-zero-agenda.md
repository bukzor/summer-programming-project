**Rewritten (user, 2026-07-11).** Two corrections to the original agenda
below drove this: (1) "before session zero" isn't a thing — session zero
*is* the before, so there's no separate pre-session-zero step; it's now
merged with the first coding sitting,
`../curriculum.kb/design/060-deliverables.kb/session-00-make-the-window-yours.md`
(one logical session 0, possibly spanning two calendar sittings if
planning runs long — see that file). (2) The old agenda pre-decided too
much on her behalf (fixed genre menu, a pgzero-vs-p5js A/B that's stale
post-pyxel-pivot, a "sprite with arrow keys" day-one target that was never
actually agreed with her). Replaced by
`../curriculum.kb/design/020-goals.kb/her-domain-stays-hers.md` and
`../curriculum.kb/design/020-goals.kb/name-the-choice-dont-pin-it.md`:
name the choice-points, don't pre-fill them.

Session zero's job, in order:

1. **Ground it.** Show a couple of minutes of a finished small crafting/
   sandbox game (doesn't have to be pyxel) so "what we're building toward"
   is concrete. Not a pitch for a specific tool — that's settled
   separately (below).
2. **Her game, her call — genuinely open, not multiple-choice.** She's
   already picked the mechanic (crafting,
   `../claims.kb/abby-chose-crafting-mechanic.md`) and lit up at a
   Minecraft-flavored frame — confirm both stand, don't relitigate them
   as if new. Everything else about the game (setting/theme, her
   character, whether there's a win condition or it stays open sandbox,
   what things are called, colors) is 100% hers
   (`her-domain-stays-hers.md`) — ask, don't offer a shortlist.
3. **Tool: shown, not re-litigated.** Show her the pyxel window running —
   this is the technical default, already decided on engineering grounds
   (`../deductions.kb/why-pyxel.md`), not a coin flip. Say so plainly, and
   say plainly that she can push back if she hates it; don't manufacture
   a false A/B choice to simulate agency that isn't real.
4. **Working agreement.** She's on the keyboard almost always (parent
   types only when asked). Session length stays 45-60 min. Cadence: state
   the aspiration (daily) and the floor (every 2-3 days,
   `../claims.kb/recommended-cadence.md`), then actually watch the first
   couple of real sessions to see what's true for her rather than
   deciding cadence in the abstract. What happens when it breaks: "we
   debug together, we don't just fix it for her."
5. **No separate day-one target.** The target *is* the session-00 Build
   that follows in the same sitting (or the next one, if time runs out):
   a window with her name and colors on it. Nothing was ever agreed with
   her about a moving sprite on day one — that claim was false and has
   been struck (`../archive.kb/claims.kb/physical-keyboard-input-unverified.md`).

Planning produces a short shared note (theme, character, win-condition-or-
not, any invented block/name she's already excited about) that session
00's Build and later sessions can refer back to — not a contract, a
starting point per `name-the-choice-dont-pin-it.md`.
