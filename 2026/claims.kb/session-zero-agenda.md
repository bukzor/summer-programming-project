Session zero (one sitting, ~30-45 min, no computer required for most of it)
should cover, in order:

1. **Show, don't pitch.** Play/watch 2-3 minutes of a couple of finished
   Pygame Zero kid-projects (or p5.js sketches) so "what we're building
   toward" is concrete, not abstract.
2. **Her game, her call.** She names: the genre (platformer, catcher,
   maze, shooter — keep the menu short, all of these are session-zero-able
   in Pygame Zero), the main character, and one sentence for "you win when
   ___."
3. **Tool choice.** Show her a Pygame Zero window and a p5.js sketch
   side by side for 60 seconds each; let her pick installed-app vs.
   browser. Don't relitigate this later.
4. **Working agreement.** Who's on the keyboard (she should be, almost
   always — parent navigates/types only when she asks). Session length
   and count per week. What happens when it breaks: "we debug together,
   we don't just fix it for her."
5. **Day-one target.** Agree out loud on the smallest visible win for the
   very first working session (e.g., "a character sprite that moves with
   arrow keys") so there's a concrete, low-bar target to hit fast.

No code gets written in session zero — its output is a one-paragraph plan
and her buy-in, not a repo.
