# Design summary

Abby (10, Scratch background) wants a Minecraft game. We want her to learn
real Python and stay hungry for the next session — the same design serves
both: a pyxel side-view Minecraft-like, built in one file, across 8 short
sessions (mine → craft → gate → build), each ending runnable and committed.

Scope is deliberately narrow: no player physics, no scrolling camera, no
drag-and-drop UI, no shaped recipes — each is high engineering cost for
near-zero beginner-Python payoff. They live in `design/070-future-work.kb/`
for if/when she wants them.

Read `design/010-mission.kb/` for the why; `design/060-deliverables.kb/` for
the session-by-session build plan most days actually need.
